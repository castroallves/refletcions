- Quando gostaria de usar um repositório remoto em vez de manter o trabalho local?

Para poder compartilhar meu trabalho



- Por que você prefere efetuar pull de alterações manualmente do que deixar o Git atualizar-se automaticamente com o seu repositório remoto?

Controlando manualmente posso ter um controle maior sobre o projeto.



- Descreva as diferenças entre bifurcações, clones e branches. Quando você usaria um e não os outros?

Clones são literalmete clones de repositórios que irão para nossa máquina, bifurcação é um tipo de clonagem mas no próprio servidor do github, branches são níveis de um repositório.
Usaria bifurcações quando quiser trabalhar diretamente associado ao repositorio originale branches para introduzir novas ideias.


- Qual é o benefício de ter uma cópia do último estado conhecido do remoto armazenado localmente?

Permite controlar o repositorio remoto enquanto trabalha no branch local e torna mais fácil a mesclagem de commits para o remoto mesclando localmente e depois enviando.



- Como você colaboraria sem usar Git ou GitHub? O que seria mais fácil e mais difícil?

Sem o git teríamos que organizar manualmente os arquivos e marcar as modificações, seria mais difícil para em um projeto
grande manter uma organização, e mais fácil para um iniciante trabalhar.


- Quando você gostaria de fazer alterações em um branch separado, e não diretamente no master? Quais são os benefícios de cada abordagem?


    Fazer alterações em um branch separado garante que o branch master estará sempre no estado de trabalho. Podemos mexer no branch separado até nos certificarmos que tudo está funcioando, já no caso do master branch seria mais fácil para trabalhar com os arquivos.
